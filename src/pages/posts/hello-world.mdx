## ✅ 학습 목적

- **동기**: 개인 블로그를 운영하며 학습한 내용을 체계적으로 기록하기 위해.
- **문제 해결**: 단순한 정적 HTML 배포 대신, 마크다운 기반으로 글을 쉽게 작성하고 자동 배포되는 구조가 필요했음.
- **목표**: GitHub Pages + Astro 조합으로 유지보수 가능한 블로그 아키텍처를 구축.

## 📖 내용 정리

### 1. 주요 개념

- **Astro**: 정적 사이트를 쉽고 빠르게 만들 수 있는 최신 프레임워크. 불필요한 JS를 줄여 빠른 로딩 속도 제공.
- **GitHub Pages**: GitHub 저장소를 이용해 정적 파일을 무료로 배포할 수 있는 서비스.

### 2. 선택 이유

- 복잡한 서버 관리 필요 없이 **정적 사이트 호스팅 가능**.
- 커스터마이징 자유도가 높은 **Astro + Tailwind** 조합.
- GitHub Actions를 활용한 자동 빌드 & 배포 파이프라인.

### 3. 앞으로의 활용

- 기술 학습 기록(TIL)과 프로젝트 회고 작성.
- 면접 준비 및 아키텍처 설계 경험 공유.
- 개발 문화/협업 노하우 정리.
